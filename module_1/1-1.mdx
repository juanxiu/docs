## 단계 1 : S3 설정과 파일 가져오기

이 단계에서는 도메인 데이터와 아티팩트를 업로드 할 두 개의 S3 버킷을 생성합니다.

### 도메인 데이터 버킷

1. Amazon S3 콘솔로 이동하여 S3 버킷을 생성합니다. 이 S3 버킷은 도메인 데이터를 저장하는 데 사용됩니다. S3 버킷 이름을 `knowledgebase-bedrock-agent-{alias}`로 지정하세요. 나머지 설정은 기본 설정을 유지합니다. 이 워크샵 전체에서 `{alias}`를 적당한 값으로 변경하는 것을 잊지 마세요.

2. 다음으로, 도메인 데이터를 다운로드합니다. [여기](https://github.com/build-on-aws/bedrock-agents-streamlit/tree/main/S3docs)에서 PDF 파일을 다운로드하세요.

파일을 한번에 다운로드 하려면 아래 링크를 이용하세요.

[파일 한번에 다운로드 하기](https://github.com/user-attachments/files/19484091/domain-dataset.zip)

3. 다운로드 받은 도메인 데이터 파일들을 S3 버킷 `knowledgebase-bedrock-agent-{alias}`에 업로드합니다.

이 파일들은 연방준비제도 이사회 회의에서 내린 통화정책 결정을 설명하는 연방공개시장위원회 문서입니다.

이 문서에는 경제 상황에 대한 논의, 뉴욕연방준비은행에 대한 공개시장운영 정책 지침, 그리고 연방기금금리에 대한 투표 내용이 포함되어 있습니다.

더 자세한 정보는 [여기](https://www.federalreserve.gov/newsevents/pressreleases/monetary20231011a.htm)에서 확인할 수 있습니다.

업로드되면 문서 중 하나를 선택하여 내용을 검토하세요.

### 아티팩트 버킷
1. 아티팩트를 저장할 다른 S3 버킷을 생성하세요. 버킷명을, `artifacts-bedrock-agent-creator-{alias}`으로 생성합니다.
그런 다음 [여기](https://github.com/build-on-aws/bedrock-agents-streamlit/blob/main/ActionSchema.json)에서 API 스키마 파일을 다운로드하세요.
<img width="1445" alt="Image" src="https://github.com/user-attachments/assets/3090d641-abe8-469c-b28c-6d778a8a4e41" />

2. 다운로드 받은 아티팩트 파일을 S3 버킷 artifacts-bedrock-agent-creator-{alias}로 업로드하세요.

제공된 API 스키마는 "포트폴리오 생성기 API"에 대한 OpenAPI 사양으로, 회사 포트폴리오 생성, 회사 재무 조사 및 이메일 발송을 위한 서비스의 구조와 기능을 설명합니다.

이 API 스키마는 각 작업에 대한 풍부한 설명을 포함하고 있어, 에이전트가 언제 사용해야 하는지, 어떻게 호출하고 결과를 사용해야 하는지 알 수 있습니다.

이 스키마는 다음과 같은 세 가지 주요 엔드포인트를 정의하고 있으며, API와 상호 작용하는 방법, 필요한 매개변수, 예상되는 응답을 자세히 설명하고 있습니다.

`/companyResearch`

`/createPortfolio`

`/sendEmail`

업로드가 완료되면 .json 문서를 선택하고 열어 내용을 검토하세요.
